03 / 08

O Motor

Detraxis Core —
o compilador de intenção.

Tratamos a geração de software como um processo de compilação em cascata. O Core orquestra dois agentes Claude distintos (Architect + Builder) para transformar a intenção do produto em um Technical Blueprint estruturado (product_definition.json) e, a partir dele, gerar a aplicação com precisão técnica.

Anatomia do motor:

  • Cascata de Enriquecimento

    O Engine Core coordena a evolução progressiva do Technical Blueprint. Ao longo da pipeline, o Architect adiciona novas definições ao product_definition.json, enriquecendo o contexto até que ele represente de forma completa a arquitetura do produto.

  • Self-Healing Loop

    Cada versão do product_definition.json é validada por JSON Schema. Quando inconsistências são detectadas, o Core aciona ciclos automáticos de correção, garantindo que o blueprint permaneça consistente antes de avançar na pipeline.

  • Pipeline Determinístico

    O Engine Core organiza a geração do software como uma pipeline estruturada baseada no Technical Blueprint. Cada etapa recebe o product_definition.json como contrato técnico, garantindo previsibilidade e consistência na geração da aplicação.

04 / 08

The Agents

Duas frentes para o Claude.

O Detraxis Core organiza o processo de geração em duas etapas complementares.

Architect

O Architect é a tecnologia da Detraxis responsável por estruturar a inteligência do processo. Em vez de pedir diretamente para o Claude gerar código, utilizamos um sistema de orquestração que guia o próprio Claude a organizar todo o contexto do produto em um arquivo chamado product_definition.json.

Esse arquivo funciona como o blueprint técnico do produto, servindo de base para toda a geração da aplicação.

Archetype

O molde estrutural

Define o tipo da aplicação (SaaS, marketplace, app de conteúdo) e orienta a arquitetura geral do produto.

Schema

O cérebro de dados

Mapeia entidades e relações lógicas que estruturam o modelo de dados e a geração de APIs.

Design Grammar

Tokens e UI

Define tokens visuais e regras de layout que garantem consistência estrutural da interface.

Skeleton

Infraestrutura pronta

Define o chassi da aplicação: runtime, autenticação, banco de dados e estrutura de projeto.

Builder

O AI Builder (Claude Code) é o agente responsável por executar o build da aplicação a partir do Technical Blueprint gerado pelo Architect.

Ele instancia o Skeleton do projeto e materializa o software definido no blueprint, conduzindo a implementação de forma autônoma até uma aplicação funcional pronta para deploy, acompanhada por um operador técnico.

Operador

O operador humano atua como controlador de voo do sistema, supervisionando a geração do software sem escrever código diretamente. Ele monitora diffs e logs técnicos em tempo real para garantir que a IA permaneça alinhada ao Technical Blueprint, corrigindo o curso sempre que algo sai dos trilhos ou apresenta inconsistências.
