01 / 05

O Pipeline

Da conversa ao MVP em 48h.
Com contratos, não com sorte.

A Deplova transforma 60 minutos de briefing em um pipeline de compilação determinístico. Cada camada passa por validação antes de avançar. Nada é gerado sem contrato técnico aprovado.

Briefing

Sessão gravada. Whisper transcreve para transcript.txt

Thinker

NotebookLM extrai estrutura, scoring de impacto e story points. Gera thinker_output.json

Sprint Planning

Cliente prioriza features em P0/P1/P2. Confirma capacidade da sprint. Exporta thinker_output_sprint.json

Bridge → Architect

Pipeline automático: blinda intenção, compila blueprint, valida 10 camadas de governança. Gera builder_handoff.json

Checkpoint

Operador revisa wireframe e contratos. Confirma ou manda regenerar.

Builder

Claude Code executa dentro do handoff. MVP no ar.

02 / 05

Os Contratos

Quatro artefatos.
Uma cadeia inquebrável.

Do briefing ao build, cada etapa produz um artefato JSON auditável que serve de contrato para a próxima.

Cliente + Operador Contrato de Sprint thinker_output_sprint.json

Features priorizadas pelo cliente com impact score, story points e capacidade da sprint confirmada.

Produzido pelo Bridge Intenção Blindada refined_intent.json

Intenção normalizada, sem HEX, sem infra não declarada. Capabilities travadas pelo Bridge.

Produzido pelo Architect Blueprint de Produto product_definition.json

Archetype, pages, features, design grammar — validado por JSON Schema com anti-hallucination.

Produzido pelo Handoff Compiler Plano de Obra builder_handoff.json

Libs concretas, injection points, protected zones, file tree map. O Builder só executa.

"O Builder não decide. Ele opera dentro de um contrato que foi auditado antes de chegar nele."

03 / 05

A Governança

Máquina governa.
Humano corrige.

Seis camadas automáticas. Dois pontos humanos. O cliente decide o escopo antes de qualquer código. O operador revisa a arquitetura antes do build. Nenhum improviso passa por aqui.

1
Sprint Planning
Cliente prioriza features em P0/P1/P2 com base em impact score e story points. Só o que o cliente confirmou entra no pipeline.
Humano
2
Sync Check
Detecta drift entre skeleton_manifest.json, bridge.py e handoff-compiler.py. Pipeline não avança com contratos desatualizados.
Automático
3
Bridge Guard
Descarta HEX, normaliza archetype, aplica restrições de infra. Só entra no pipeline o que o skeleton consegue construir.
Automático
4
Schema Validation
O product_definition.json é validado contra JSON Schema com additionalProperties: false. O Architect não pode inventar campos.
Automático
5
Governance Audit
Identity check, capabilities lock, DB access guard, feature coverage P0. O Architect não pode mudar o que o Bridge definiu.
Automático
6
Coherence Check
Skeleton suporta as capabilities? Rotas conflitam com protected zones? Viabilidade técnica antes do handoff.
Automático
7
Handoff Compiler
Traduz capabilities abstratas em libs concretas, rotas em caminhos de arquivo, injection points em instruções operacionais. O Builder não interpreta — só executa.
Automático
8
Checkpoint do Architect — Corregedoria
Operador revisa blueprint e wireframe de baixa fidelidade. Confirma ou manda regenerar. Pipeline pausa aqui — o Builder não começa sem autorização humana.
Humano
Checkpoint em ação
Architect
Blueprint gerado
Pipeline pausa

04 / 05

A Prova Técnica

Não é prompt engineering.
É um compilador.

A diferença entre "pedir pra IA" e o Deplova é a mesma diferença entre escrever código em texto livre e usar um compilador tipado. O compilador rejeita o que não obedece o contrato.

run-pipeline.bat
[0/4] Sync Check
✅ Skeleton sincronizado

[1/4] Bridge
✅ refined_intent.json gerado
✅ 4 features validadas

[2/4] Architect
✅ Blueprint compilado

[3/4] Governance | Coherence | Handoff
✅ Schema validado
✅ Capabilities locked
✅ Identity consistente
✅ DB BLOCKED respeitado
✅ Skeleton fit confirmado
✅ Sem conflito de rotas
✅ reasoning_trace.json salvo

🎉 BINGO — Builder autorizado.
Anti-hallucination
O Architect não pode inventar

additionalProperties: false no schema. Qualquer campo fora do contrato rejeita o output inteiro.

Capabilities Lock
Nenhum agent muda o escopo

As capabilities são definidas no Bridge e travadas. O Architect compila — não decide. O Builder executa — não interpreta.

Determinismo
Mesmo input, mesmo output

Pipeline fail-fast: qualquer gate quebrado para a execução antes de avançar. Nenhuma etapa começa sem a anterior aprovada.

Rastreabilidade
O Architect documenta suas decisões

reasoning_trace.json registra por que cada archetype, navegação e página foi escolhida. Degradações aplicadas ficam documentadas. Debug e auditoria em segundos.

05 / 05

Oportunidade e Visão

De execução manual para infraestrutura de geração de produto.

Mês 0: Core Validado

Pipeline determinístico com validação de schema e coerência física.
Resultado: Garantia de blueprints consistentes e prontos para produção.

Meses 1–3: Prova de Valor (Tração)

Implementação em projetos reais para refinamento de padrões.
Resultado: Redução do lead time de desenvolvimento e consolidação do feedback de campo.

Meses 4–6: Automação Avançada (Escala)

Evolução da autonomia do Core e expansão da cobertura de casos de uso.
Resultado: Minimização da intervenção humana e aumento da margem operacional.

Meses 6–12: Ecossistema Deplova (Plataforma)

Transição de ferramenta para plataforma de governança e escala.
Resultado: Geração contínua de produtos com padronização industrial.
